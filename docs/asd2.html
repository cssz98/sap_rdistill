10.6.1.Servlet osztály létrehozása

10.6.1.1.DEF

-  Egy speciális Servlet típusú osztály, ami a Glassfish szerverre kívülről hívható Servleteket telepít adott URL-re (itt most localhost)

-  Amiknek segítségével a frontend AJAX-ból meghívhatja a Service csomag CRUD+A metódusait

-  JSON-on keresztül kommunikálva (vagy pl. XML-ben):

-  Request: Frontend->Szerver:

-   Egyezményesen „task” kulccsal jön a service CRUD+A neve, amit meg kell hívni

-   A bemenő paramétereik a paraméter nevével egyező kulccsal jönnek

-  Response: Szerver->Frontend:

-   C, U, D egyezményesen „msg” kulccsal siker/sikertelenség jelzést ad szöveggé konvertált JSONObject-ben

-   R és A szöveggé konvertált visszatérő JSONArray-t ad ki

-  Nem tárgyaljuk, hogyan lehet azon URL-eket korlátozni, honnan érhetik el a servletet, itt most mindenhonnan elérhetik

10.7.2.Egy adatbázis táblához/Entity Class-hoz tartozó JavaScript CRUD+AF függvények írása

10.7.2.1.Cél

-  A modern frontend programozás célja, hogy minimalizálja az oldalra rakandó statikus elemeket, és szinte mindent dinamikusan hozzon létre, ezáltal biztosítva a kód újrafelhasználhatóságot különböző oldalak közt

-  Ezért a CRUD+A függvények írásánál az a cél, hogy az oldalon statikusan egyetlen kopasz, üres <Table ></Table> elemre legyen hozzájuk szükség, ráadásul azt is paraméterezzük, milyen nevű Table-be dolgozzanak

-  A tábla elrendezésével biztosítjuk, hogy minden CRUD+A kezelő elem elférjen benne:

10.7.2.2.Általános függvényszerkezeti séma

-  JavaScript nem szigorúan típusos: változókat var-al kell deklarálni, de nem muszáj a típusukat megadni, bármely típust bármely típusba autokonvertál, ha tud

-  JavaScript nem objektum orientált (majd későbbi verziója a TypeScript lesz az), csak struktúrált, így csak oldal betöltődés előtt lefutó főprogram meg függvények vannak benne:

-  function kulcszóval kezdődik, majd a CRUD+AF neve jön

-  Bemenő paraméterlistája nem tipizált, és akkor üres, ha fel tudja adatokat felszedni valamilyen HTML-elemből

-  Visszatérő értéke nem tipizált, és nem kötelező, hogy legyen

-   Ha HTML elemekből szed fel bemenő adatot, akkor az legyen egy id=osztalynevMezonev -vel hivatkozott form, és abban legyen egy azonos nevű input elem (mert az input id-vel hivatkozása problémás). var-al változót deklarálunk a form-ra és .elements[0].value-vel szedhetjük ki belőle az értéket

-   Ezt használjuk egy szervernek küldendő, kuldendo nevű JSON összeállítása során a „mezonev”: kulcsokkal jelőlt paraméterek értékadására. A JSON-ból nem felejtjük ki „task”: kulccsal beadni a Servleten megírt task nevét.

-  Ezután $.ajax({ fejléccel összeállítjuk az AJAX-hívást:

-   url: „osztalynevServlet”, //servlet meghívása

-   type: „post”, //Post típusú üzenet legyen, vagyis a webkereső címsorában ne látsszon a JSON tartalma

-   data: kuldendo //Összeállított JSON szerverre küldése

-   success: function(vissza) { … } //Mit kell tenni sikeres szerver válasz esetén, ahol a vissza nevű JSON-ban jönnek az adatok szerverről:

-   alert(vissza.msg); //Fejlesztési időszakban mindig adjak webkereső alert-be siker esetén is jelzést, hogy tudjam, mi történt meg

-   vissza[index].mezonev //szerverről jövő rekordok mezőértékeit megadhatom HTML elemek .innerHTML vagy .value tulajdonságának, megjelenítve őket

-   error: function(vissza) { … } //Mit kell tenni sikertelen szerver válasz esetén, ahol a vissza nevű JSON-ban jönnek az adatok szerverről.

function getJatekos() {

    var jatekosID = document.getElementById(“jatekosID”);

    var jatekosNev = document.getElementById(“jatekosNev”);

    var jatekosSzulEv = document.getElementById(“jatekosSzulEv”);

    var jatekosGolok = document.getElementById(“jatekosGolok”);

    var jatekosNem = document.getElementById(“jatekosNem”);

    var jatekosCsapatID = document.getElementById(“jatekosCsapatID”);

    var kuldendo = {“task”: “getJatekos”, “jatekosID”: jatekosID.elements[0].value};

    $.ajax({

        url: “JatekosServlet”,

        type: “post”,

        data: kuldendo,

        success: function(vissza) { <!—Szerver vissza válaszból felnyomja formok inputjaiba az adatokat –>

            alert(“Játékos keresése:” + jatekosID.elements[0].value);

            jatekosID.elements[0].value = vissza[0].jatekosID;

            jatekosNev.elements[0].value = vissza[0].nev;

            jatekosSzulEv.elements[0].value = vissza[0].szulEv;

            jatekosGolok.elements[0].value = vissza[0].golok;

            jatekosNem.elements[0].value = vissza[0].nem;

            jatekosCsapatID.elements[0].value = vissza[0].csapatID;

        },

        error: function(vissza) {

            alert(“Kapcsolati hiba játékos keresésénél:” + jatekosID.elements[0].value);

        }

    });

}
